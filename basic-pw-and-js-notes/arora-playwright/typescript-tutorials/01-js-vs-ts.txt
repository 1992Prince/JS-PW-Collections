üîπ JavaScript Example (No Type Safety)

// example.js
function add(a, b) {
  return a + b;
}

console.log(add(10, 20));       // ‚úÖ 30
console.log(add("10", 20));     // ‚úÖ "1020" (string concatenation, no error!)
console.log(add(true, null));   // ‚úÖ 1 (no compile-time error!)
‚ùå JavaScript does not warn if incorrect data types are passed ‚Äî it just runs!

üîπ TypeScript Example (With Type Safety)
// example.ts
function add(a: number, b: number): number {
  return a + b;
}

console.log(add(10, 20));         // ‚úÖ OK
console.log(add("10", 20));       // ‚ùå Compile-time Error: Argument of type 'string' is not assignable to parameter of type 'number'
‚úÖ TypeScript prevents errors at compile time, ensuring type correctness.

üî∑ 1. Definition
JavaScript	
A scripting language used to create dynamic web content.	

TypeScript
A superset of JavaScript that adds static typing.

üî∑ 2. Typing System
| JavaScript                                             | TypeScript                                                |
| ------------------------------------------------------ | --------------------------------------------------------- |
| **Dynamically typed** ‚Äì types are resolved at runtime. | **Statically typed** ‚Äì types are checked at compile time. |
| You can write: `let name = 123;`                       | You can write: `let name: string = 'John';`               |

üî∑ 3. Compilation
| JavaScript                           | TypeScript                                                                    |
| ------------------------------------ | ----------------------------------------------------------------------------- |
| Interpreted directly by the browser. | Needs to be **compiled to JavaScript** using the TypeScript compiler (`tsc`). |

üî∑ 4. Error Detection
| JavaScript                    | TypeScript                                                   |
| ----------------------------- | ------------------------------------------------------------ |
| Errors appear **at runtime**. | Errors are **caught during development** (before code runs). |

üî∑ 6. Use in Projects
‚úÖ JavaScript is suitable for small scripts or when flexibility is needed.

‚úÖ TypeScript is used for large-scale applications, especially where maintainability and collaboration are important 
   (e.g., Angular, Playwright, NestJS projects).
   
   üß† Interview Tip:
   ‚ÄúJavaScript is dynamically typed and flexible, which is good for small-scale apps or quick prototypes. 
   TypeScript, on the other hand, brings in static typing, early error detection, and better developer tooling, 
   making it ideal for large and maintainable codebases. 
   Since TypeScript is a superset of JavaScript, all JS code is valid TS, but TS adds optional type safety which 
   boosts productivity and reduces bugs.‚Äù
