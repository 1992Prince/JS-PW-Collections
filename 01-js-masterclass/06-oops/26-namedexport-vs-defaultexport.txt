ğŸ”¹ Named Export
-------------------------------------------------------------------------

ğŸ‘‰ Curly braces mandatory hote hain import ke time.
ğŸ‘‰ Hamesha same naam se import karna hota hai (except jab alias dete ho).
ğŸ‘‰ Multiple functions/classes ko export kar sakte ho.



Example (functions + aliasing)

ğŸ“ mathUtils.ts

// Named exports
export function add(a: number, b: number) {
  return a + b;
}

export function subtract(a: number, b: number) {
  return a - b;
}


ğŸ“ app.ts


// Curly braces mandatory âŒ without them won't work
// below we are importing add fun with same name but subtract fun with alias 'minus'
import { add, subtract as minus } from './mathUtils';

console.log(add(10, 5));   // 15
console.log(minus(10, 5)); // 5  (alias used)




Example (multiple classes)

  radius: number;

ğŸ”¹ Default Export
-------------------------------------------------------------------------

ğŸ‘‰ Ek hi default export hota hai per file.
ğŸ‘‰ Import karte waqt {} use nahi hote âŒ.
ğŸ‘‰ Naam apne hisaab se kuch bhi rakh sakte ho.

Example (default export function)

ğŸ“ logger.ts

// Default export function
export default function log(message: string) {
  console.log("LOG:", message);
}

ğŸ“ app.ts

// {} nahi lagte default export ke liye
import myLogger from './logger';

myLogger("Hello World"); // LOG: Hello World



Example (default export class)

ğŸ“ User.ts

// Default export class
export default class User {
  name: string;
  constructor(name: string) {
    this.name = name;
  }
}


ğŸ“ app.ts

import Person from './User';

const u = new Person("Rahul");
console.log(u.name); // Rahul



ğŸ”¹ Named Export vs Default Export (Tabular Comparison)

| Feature             | Named Export âœ…                    | Default Export âœ…                                         |
| ------------------- | --------------------------------- | -------------------------------------------------------- |
| **Curly braces**    | Required (`import { add }`)       | âŒ Not used (`import log`)                                |
| **Number per file** | Multiple allowed                  | Only one default per file                                |
| **Import Name**     | Must match export name            | Can be renamed freely                                    |
| **Alias**           | Supported via `as` keyword        | Direct rename during import                              |
| **Use Case**        | When file exports many utilities  | When file has one main entity                            |
| **Examples**        | `export function...` / `class...` | `export default function...` / `export default class...` |


ğŸ”¥ One-liner for Interview

"Named exports require curly braces, allow multiple exports per file, and names must match (or use alias), 
while default export has only one export per file, doesnâ€™t use curly braces, and can be imported with any name."
