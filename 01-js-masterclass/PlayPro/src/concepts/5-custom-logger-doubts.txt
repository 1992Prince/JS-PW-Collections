Loggers:

PW itself don't provide any built-in logger. 
So we need to come up with custom logger and define info, warn, debug etc.

ðŸ”¹ Why we need loggers in test automation framework?

- Better Debugging

        Agar test fail hota hai to console output usually limited hota hai.

        Logger se detailed step-by-step information milti hai 
        (which step executed, what data used, etc.).

- Traceability

        Kis time pe kaunsa step execute hua, kaunsa API call hua, ya kaunsa element interact 
        hua â€“ ye sab logs me track kar sakte hain.

- Error Diagnosis

        Failures ke waqt sirf stack trace se samajhna mushkil hota hai.

        Logs bata dete hain ki failure ke pehle tak kaunsa step successfully run hua tha.

- Custom Levels of Information

        Loggers ke through alag-alag levels set kar sakte hain:

        INFO â†’ General test execution steps

        DEBUG â†’ Detailed technical info (for developers)

        WARN â†’ Possible risky steps

        ERROR â†’ Failures or exceptions

- Persistent Record

        Console logs run ke baad khatam ho jaate hain, but logger ke outputs file me store kiye 
        ja sakte hain.

        Ye CI/CD pipeline me bhi kaam aata hai (Jenkins, GitHub Actions, etc.).

- Helps in CI/CD & Team Collaboration

        Agar tests Jenkins/GitHub Actions me run ho rahe hain, to logs se team easily samajh 
        paati hai ki kahan aur kyun fail hua.

- Readable Test Reports

        Loggers ko reports (like Allure, HTML reports) ke saath integrate karke readable execution 
        history milti hai.


ðŸ”¹ Where loggers help?

    During local debugging â†’ Jab developer ya tester apne system pe run karta hai.

    During CI/CD pipeline execution â†’ Jab test cloud/VM pe run ho rahe hote hain aur tumhe sirf logs milte hain.

    During root cause analysis â†’ Jab production bug reproduce karna hai to logs help karte hain.

    During maintenance â†’ Framework ya test update karte waqt, logs help karte hain to identify regression failures.

private timestamp(): string {
    return new Date().toISOString();
}

- Return type: string
- new Date()

    JavaScript ka built-in Date object.

    new Date() se current date & time milega (system clock se).

    Example:
    console.log(new Date());
    // Output: 2025-08-28T12:34:56.789Z  (depends on system time)


- .toISOString()

    Date ko ISO 8601 format me string bana deta hai.
    
    Format: YYYY-MM-DDTHH:mm:ss.sssZ
    
    T â†’ separator between date and time
    
    Z â†’ UTC timezone (zero offset)

    Example:
    console.log(new Date().toISOString());
// "2025-08-28T12:34:56.789Z"



ðŸ“Œ Explain Method Code

info(message: string, ...args: any[]) {
    console.info(`\x1b[34m[${this.timestamp()}] INFO: ${message}`, ...args);
}


Parameters - (message: string, ...args: any[])

    message: string

    Main log message (string type hona compulsory hai).

    ...args: any[]

    Ye rest parameter hai.

    Matlab tum is function ko call karte waqt extra arguments pass kar sakte ho.

    args ek array ban jaata hai jo sab extra values ko hold karta hai.

    Baaki jitne bhi arguments pass karoge, wo ek array ban jaayenge args ke andar.

ðŸ‘‰ Example:

    args = [ { id: 101, name: "Rahul" }, "from mobile" ]
