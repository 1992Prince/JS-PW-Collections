üëâ Question:
Why do we need JSON.stringify() when printing a JSON object, but not when printing a JSON array?

‚úÖ Answer

JavaScript ka console.log aur string interpolation (${...} inside template string) alag behave karte hain:

1. Object interpolation

const obj = { env: "qa", timeout: 5000 };
console.log(`obj is - ${obj}`);

Output:

obj is - [object Object]

üëâ Reason:

Jab tum object ko template string ke andar ${obj} likhte ho, JS us object ka toString() method call karta hai.

Aur plain object {} ka default toString() hota hai: [object Object]

Isliye aisa output milta hai.

Agar tum JSON.stringify(obj) karte ho, to object ko proper JSON string me convert kar deta hai:

console.log(JSON.stringify(obj)); 
// {"env":"qa","timeout":5000}


lekin if we print same object directly in console.log without template string then
console.log(obj); // yeh sahi se print karega object ko i.e.. { env: "qa", timeout: 5000 }


2. Array interpolation

const arr = [101, 102, 103];
console.log(`arr is - ${arr}`);

Output:

arr is - 101,102,103

üëâ Reason:

    - Array ka default toString() method numbers ko comma-separated string bana deta hai.

    - Matlab arr.toString() return karega "101,102,103".

    - Isliye bina JSON.stringify() ke bhi readable output milta hai.

Lekin agar tum JSON.stringify(arr) karoge:

console.log(JSON.stringify(arr));
// [101,102,103]

To ye JSON-compliant representation deta hai (brackets ke saath).

üîπ What is JSON.stringify()?

JSON.stringify() ek JavaScript function hai jo kisi bhi JS value 
(object, array, number, string, etc.) ko JSON string me convert karta hai.

Syntax: JSON.stringify(value, replacer?, space?)


üîπ Purpose

1. Object ya Array ko human-readable / transferable string me convert karna

    - JS object ko directly log karoge to "[object Object]" milega using interpolation way
      [If we are just using console.log(obj) then it will print the object properly]

    - JSON.stringify() usko valid JSON string bana deta hai.

2. Data ko file, API, ya database me save / transfer karna

    - File systems, APIs, databases ko string chahiye, JS objects nahi.

    - JSON.stringify() use karke tum object ko string bana ke bhej sakte ho.

  
üîπ Example
‚úÖ Without stringify

const user = { name: "Rahul", age: 25 };
console.log(user);          // { name: 'Rahul', age: 25 }
console.log(`${user}`);     // [object Object]

‚úÖ With stringify
console.log(JSON.stringify(user)); 
// {"name":"Rahul","age":25}



üîπ Use Cases
1. Logging / Debugging

const config = { env: "qa", timeout: 5000 };
console.log("Config:", JSON.stringify(config));

üëâ Clean readable output.

2. Storing in Local Storage (Browser)

const user = { name: "Rahul", loggedIn: true };
localStorage.setItem("user", JSON.stringify(user));  // Save as string

const savedUser = JSON.parse(localStorage.getItem("user")!); // Parse back to object
console.log(savedUser.name); // Rahul
