Q: What is a frame or iframe in HTML?

1. Frame ka object kaise milta hai?

a) By name attribute

<iframe name="frame-login" src="https://example.com/login"></iframe>

const frame = page.frame('frame-login');   // frame name se access
await frame.fill('#username-input', 'John');


b) By URL

<iframe src="https://myapp.com/login"></iframe>

const frame = page.frame({ url: /.*myapp\.com\/login.*/ }); #const frame = page.frame({ url: /.*domain.*/ });
await frame.fill('#username-input', 'John');

ğŸ‘‰ Yahaan regex (/.*myapp\.com.*/) use karke URL match kar raha hai.





ğŸ‘‰ "Agar page.frame() se frame access ho hi raha hai, toh frameLocator kyon aaya Playwright me?"

ğŸ”¹ 1. page.frame() kya karta hai?

    - Ye ek frame object return karta hai (iframe ke andar ka pura page).
    - Uske baad tumko manually us frame me jaake locator likhna padta hai.

    const loginFrame = page.frame('frame-login');  // frame object lo
    await loginFrame.fill('#username', 'john');    // uske andar ka element fill

ğŸ”¹ 2. frameLocator kya karta hai?

- Ye auto-waiting + auto-resolution deta hai.
- Tum directly iframe ke andar ke elements target kar sakte ho bina frame object manually handle kiye.
- Matlab â†’ tum locator ko chain kar sakte ho, aur Playwright khud dekh lega ki iframe kab load hota hai.

    // directly iframe ke andar locate
    await page.frameLocator('#login-frame')
        .getByRole('textbox', { name: 'Username' })
        .fill('john');



3. Difference in Short

| **Feature**             | **page.frame()**             | **frameLocator()**                               |
| ----------------------- | ---------------------------- | ------------------------------------------------ |
| How it works            | Gets frame object manually   | Auto-locates elements inside iframe              |
| Reload / dynamic frames | Can break (old ref)          | Always works (auto-resolves)                     |
| Waiting                 | You handle it                | Built-in auto-wait                               |
| Locator chaining        | Not possible                 | Possible (frameLocator â†’ getByRole â†’ click etc.) |
| Recommended?            | âœ… Sometimes (advanced cases) | â­ Preferred for normal element actions           |


ğŸ¯ Interview Style Answer

"Playwright provides two ways to interact with iframes. With page.frame(), 
you explicitly get the frame object and then work inside it, but this can break if the iframe reloads or 
changes dynamically. frameLocator is the recommended modern approach because it supports 
locator chaining, auto-waiting, and automatically resolves elements inside the iframe. So, in most cases, 
frameLocator is more reliable than page.frame()."
