âœ”ï¸  Why do we need tsconfig.json in TypeScript?

    1) TypeScript needs to be compiled:

        TypeScript is a superset of JavaScript with types.
        Browsers and Node.js cannot run TypeScript directly.
        tsconfig.json guides the compiler on how to convert .ts files into runnable .js.


    2) Control over compilation behavior:

        You decide target environment, strictness, module formats.
        Helps ensure your code is compatible with the environment you want.

    3) Project structure:

        You specify which files are part of the project.
        Helps large projects organize code and output.


âœ”ï¸  Why donâ€™t we need it in JavaScript?

        JavaScript is interpreted directly by browsers or Node.js.
        No compilation step is needed (unless you use a bundler or transpiler like Babel).
        No need to configure compilation options for plain JS.
